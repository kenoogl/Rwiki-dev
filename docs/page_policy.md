# AGENTS/page_policy.md

## Purpose

Define wiki page types and how to choose the correct destination for new or updated pages.

## Types

### concepts
Core ideas, theories, and explanatory pages.
Primary focus: what it is / why it matters.

### methods
Implementation, procedures, algorithms, and technique comparisons.
Primary focus: how it works / how to use it.

### projects
Repository- or research-project-specific goals, decisions, status, and open issues.

### entities
Named external objects such as people, tools, datasets, and organizations.

### synthesis
Approved reusable conclusions, design patterns, and cross-cutting insights that are not just single-topic explanations.

## Selection Rule

- Choose the page type by primary purpose, not by topic name
- If a topic could fit multiple types, prefer the type that matches the page’s main use
- Prefer updating an existing page over creating a new page in a different type
- Do not create a new page if the content can be integrated into an existing one
